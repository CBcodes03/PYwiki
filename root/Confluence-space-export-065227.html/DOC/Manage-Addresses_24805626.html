Documentation : Manage Addresses

Created by Siddharth Rastogi, last modified on Mar 25, 2018

This explains how the user will follow if he wants to either add or change an address.

This flow contains basically two pages:

a. My Address List (to switch delivery location or edit an address)

b. Add New Address (for adding a new address to the address list)


a. My Address List

This page will open when user clicks on the "Change Address" button on the Checkout Page. This page will list down all the addresses ever added by the user.

Data Points and Logics

S.NoData Points / ButtonsLogic
1Add New ButtonThis button appears on the toolbar towards the right. On click, open teh Add New Address Page (as explained below)
2Radio buttons

User can select any one address at any given time.

There must be at least one selected address at any given time.

3Address List

Each Address Row must display the address in following format:

  • Full Street name (In bold)
  • City Name - Pincode,
  • State Name,
  • Reg Phone No.
  • Optional Phone No. (if provided)
4Edit Button 

On click, the "Add New Address" page will open with its Title changed to "Edit Address".

All the data on the page will be prefilled and user can edit and update the Address from there.

5Continue ButtonOn click, check the address whose radio button is selected and mark that address as "selected" in the Database. On click, close the page and open the Checkout page where the selected address should be populated.




b. Add New Address

This page can be accessed from three points:

a. Add New Address Button on checkout page in shipping section

b. Add New Button from the My Address List Page (as described above)

c. Edit Button from the My Address List Page 

This page will be used for both "adding" or "updating" an address.

Data Points and Logics 

S.NoData Points / ButtonsLogics
1Page Title 

"Add New Address". 

If page is opened via "Edit" button from My Address List page, then change the page title to "Edit Address"

2Registered Phone No.Populate the registered no. of the user. This field be in a locked state. 
3Pin Code

On click, open a small popup for entering the pin code.

Automatically open the Keyboard (with number inputs only).

Validate this edit box with 6 Digit input. User should not be able to enter more than 6 Digits. If a user tries to enter less than 6 digits and clicks on Save Button, show toast "Please enter a valid Pincode"

When a user enters a pin code and clicks on Save Button, Run the pincode checker API:

  • if pincode is serviciable, close the popup box and populate the Pincode box on the page and show the location values below the Pincode Edit Box in the following format: Your Location: 'City Name', 'State Name'
  • If pincode is not serviciable, show the toast "This pincode is not in our service area".
4Street AddressHere user can enter the street address. Validate this field for minimum 10 Characters. 
5Secondary Phone This field is optional. Open keyboard (number input). 10 digit validation
6Save Address Button

On click, Save the address above and close this page. 

Now if the user had landed on this page from "Add New Address" button on Checkout page, then on click of save button, open the Checkout page directly.

Whereas if the user had landed on this page via "Add New" Button or "Edit Button" of the My Address List Page, then on click of save button, open the My Address List Page.
