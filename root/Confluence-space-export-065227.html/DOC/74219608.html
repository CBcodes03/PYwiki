Documentation : Buyer Verification (Updated)

Created by Siddharth Rastogi, last modified by Quality Assurance Sk on May 20, 2018

Overview

This screen will be used to verify the buyers whose documents are not verified.  This page will list down all the buyers whose have or have not given any form of verification doc and are not verified. 

Overall, admin can do 3 actions here:

  • Verify the Buyer
  • Cancel Orders
  • Change the Address


Logic to show Buyers

List the buyers based on the following logic:

For every buyer check, 'tinNo' field in user_master table.

  • If Empty, check 'orderVerifyDone'
    • If orderVerifyDone > 0, then do not show the buyer entry
    • If orderVerifyDone = 0 AND (COUNT o.orderid = 0 WHERE o.status = 4), then do not show the buyer entry
    • If orderVerifyDone = 0 AND (COUNT o.orderid > 0 WHERE o.status = 4), then show the buyer entry
  • If Not Empty, check 'isGSTVerified'
    • if isGSTVerified > 0, then do not show the buyer entry
    • if isGSTVerified = 0 AND (COUNT o.orderid = 0 WHERE o.status = 4), then do not show the buyer entry
    • If isGSTVerified = 0 AND (COUNT o.orderid > 0 WHERE o.status = 4), then show the buyer entry

By default the list of buyers will be according to the "um.verifReqOn" field (Descending) - Field yet to be added


  1. How will i show the buyers who have not given any documents for verification but have placed an order.
  2. The above logic will not show any buyers if the GST is verified and buyer has uploaded any verification document to be verified.

This page will have the standard sort and Search features. (Do not show the Filters and Download CSV button)

The sort button will toggle between ASCENDING & DESCENDING order using um.verifReqOn field.

Data Points and Logic

#Data Points / ButtonsLogic
1Total Buyers

This appears below the Search section. This will show the total count of buyers that appear on this page. (using logic as explained above).

2PaginationThis is the standard pagination used for all other pages.
3R / N icons 

These icons appear before the Buyers name in Order Card. R stands for Repeat Buyer and N Stands for New Buyer.

For any order only one of these icons will be present at a given time. 

Logic to show Repeat/New icons

Show R Icon: if the buyer has placed 2 or more orders and atleast one order has o.status=4 AND o.confirmedOn>0

Show N icon: in else condition of the above logic.

Keep orange color for R and Cyan color for N.

4Buyer Name

The 'companyName' of the buyer who has placed the order.  [um.companyName]

Maximum character limit of 45. If exceeds show .... SB-331 - Data cannot be retrieved due to an unexpected error.

5Buyer Phone No.This appears just below the Buyer Name. [um.companyPhone]
6Verification info 

This section appears to the right of the buyer name. There will be two labels:

  1. Document Verification
  2. GST Verification

Each label will be appended by either Red exclamation icon, Green Tick icon or 'Not Provided' text on the basis of the following condition.

Logic for verification icon

For Document Verification label,

  • Show Red Exclamation icon if, orderVerifyDoc is not NULL AND orderVerifyDone = 0
  • Show Green Tick icon if, orderVerifyDoc is not NULL AND orderVerifyDone > 0
  • Show 'Not Provided' text if, orderVerifyDoc is NULL 

For GST Verification Label, 

  • Show Red Exclamation icon if, tinNo is not NULL AND isGSTVerified=0
  • Show Green Tick icon if, tinNo is not NULL AND isGSTVerified > 0
  • Show 'Not Provided' text if, tinNo is NULL 
7Four Tabs

Each Order Card will have four tab menus. 

  • Verification
  • Buyer Details
  • Orders
  • Comment Log (x) ; where X is the no. of comments (These comments will be stored and fetched from user_master table)

On clicking any menu, the card will expand to their respective Ui's. On clicking again on an already opened menu tab, will collapse the card to its default Ui. 

8Verification Tab

This Tab contains two sections:

  • Document Verification
  • GST Verification

(Each Explained Below)

9Document Verification

This section has following items:

  1. Verification Type : This will show a dropdown showing the values selected by the buyer from the app. ("um.orderVerifyType" from user_master table). On clicking the dropdown a list of options will appear: Bill Book, Visiting Card and Shop Photo. On selection of any option it will set the value in the dropdown. (do not update in database at this stage). If the "um.orderVerifyType" field is NULL then show the empty box. If the um.orderVerifyDone>0, then the dropdown will be locked and cannot be changed.
  2. Verification info: If the 'um.orderVerifyDoc' field is not NULL, show a hyperlinked text "View Image" on click of which open the url file in a new tab. In case the 'orderVerifyDoc' field is NULL, show a text "No Image"
  3. Upload button: On click, open file upload window. Save the file and update the file URL in [um.orderVerifyDoc]  (only PNGs and JPEGs). The upload button will be disabled when "orderVerifyDone>0"
  4. Checkbox - This check box conditions are:
    1. Show checked if "orderVerifyDone > 0"
    2. Show unchecked if "orderVerifyDone = 0".
    3. If "orderVerifyDone > 0 " then the lock the checkbox such that it cannot be unchecked in Ui.
    4. If "orderVerifyDone = 0" then the admin can toggle check/uncheck on this checkbox (Do not update Db at this stage)
  5. Ui Check: Checkbox will be disabled if "orderVerifyType IS NULL" OR "orderVerifyDoc IS NULL" 
10GST Verification

This section has following items:

  1. GST Number: this will show an edit box which will fetch values from 'tinNo' from user_master table. The GST No Edit box will be locked when "isGSTVerified>0"
  2. Checkbox - This check box conditions are
    1. Show checked if "isGSTVerified > 0"
    2. Show unchecked if "isGSTVerified = 0".
    3. If "isGSTVerified > 0" then the lock the checkbox such that it cannot be unchecked in Ui.
    4. If "isGSTVerified = 0" then the admin can toggle check/uncheck on this checkbox (Do not update Db at this stage)
  3. Ui Check: Checkbox will be disabled if "tinNo IS NULL" or not in correct format of GST
  4. GST input format validation: 2 Digits Followed by 5 English Characters followed by 4 digits followed by 1 character followed by 1 digit followed by 1 character followed by 1 digit

12Shipping Address Popup

This popup will show the pre-filled address from the address_details table of the buyer WHER "isSelected=1". This popup will be used to edit the address. On Saving the address the respective fields must get updated.

  • Street Address (ad.companyAddress1) 
  • Pincode (ad.zipcode)
  • Check Availability Button - On click, run the pincode servicibility api used in the app and find if the pincode is available or not. If available, show green tick, if not available show red exclamation mark.
  • City and State fields will be locked edit boxed. By default they will be prefilled and fetched from the ad.city and ad.state from address_details table. In UI, they will be updated by the servicibility api response.
  • Update Button: on click, Check the Servicibility api for pincode abailability. If yes, update the ad.pincode, ad.city, ad.state and ad.companyAddress1. If Not, show exclamation mark next to pincode edit box. 
13Orders menu tab

This will expand and show the list of orders where: (o.remainingPairs > o.completedPairs AND o.confirmedOn > 0) OR (o.status=4 AND o.confirmedOn=0)

  • Name of Seller [um.companyName using o.sellerID]
  • Seller's city [ad.city using o.sellerID WHERE ad.isPickup=1]
  • Order ID [o.skOrderid]
  • Order Placed Date [o.updatedOn]
  • Status:
    • Show text "Running Order" if  (o.remainingPairs > o.completedPairs AND o.confirmedOn > 0); Text color #FF9C4B
    • Show text "Confirmation Pending" if (o.status=4 AND o.confirmedOn=0); Text Color #26C6DA
  • Net Amount [o.netAmount]
  • Total SKU [o.items]
  • Total pairs [o.pairs]

On click of any order entry row, open "Order Items" Pop Up (Explained Below) 

14Order Items PopUpThis popup will open the UI of Order Items Tab from the "All Orders Screen". All the logics and conditions applied there will be replicated here as well.
15Comment log

The Comment Log text will be appended by "(X)" ; where X = no. of comments on that user. 

On click, open the comment log popup (as designed by Pratyush). This will fetch and update data from the user_master table.

16Cancel Order

This button will appear on the left side of "Mark verify" button only in following conditions:

"isGSTVerified = 0" AND "orderVerifyDone = 0" AND (COUNT o.orderid > 0 WHERE o.status = 4)

On click, set o.status=5 for all orders WHERE o.confirmedOn = 0

17Mark Verified

Case 1 : Check if Document Verified checkbox is enabled (orderVerifyDone=0) & marked checked in Ui, then update following fields in DB

  1. orderVerifyType
  2. orderVerifyDoc
  3. orderVerifyDone (timestamp)

Case 2 : Check if GST Verification checkbox is enabled(isGSTVerified=0) & marked checked in Ui, then update the following fields in Db

  1. tinNo
  2. isGSTVerified (timestamp)
