Documentation : In Packaging (old)

Created by Siddharth Rastogi, last modified on Dec 10, 2018

Overview

This page will basically be used for browsing all the items items that have been moved to packaging by the seller. This page will be divided into two parts:

a. In Packaging

b. Initiated Pickup

This page will have 4 major actionable:

  1. Initiate Pickup
  2. Cancel Items
  3. Move back to In Process

In Packaging

4Initiate Pickup Button

On click of this button a small dialog will open having inputs of “Count of Cartons” & “Invoice Number”.

The validations of invoice No: One seller cannot use an invoice no. twice.

On clicking “OK” button from the dialog, following things to be done:

  1. Mark the Consignment as Initiated Pickup
  2. Save the GST no. of the Buyer and Seller at consignment level (if any)
  3. Run checks with the Logistic Partner APIs for pin code serviceability.
  4. Update following info such as: Consignment Value, Shipping, Tax, Discounts advance payable etc.
  5. Save the Request for manifestation and response in the DB.
  6. Call the manifestation API
  7. Do not map the Consignment to the payment cycle at this stage.
  8. Save the B2D Delay value if the delay is positive in order details table. Logic: Update the Value "(CurrentTimeStamp - od.expectedDispatchDate)" if (CurrentTimeStamp > od.expectedDispatchDate)
  9. Check if buyers' um.isGSTVerified > 0. If yes then update the cm.buyerGSTNo (New field) with the value of um.tinNo

5Cancel All Items

This will open the Cancellation Popup (Without the Pairs Selection dropdown). When Reason is selected and confirmed, mark all the lots having status “In packaging” as cancelled.

On Click of "Confirm Cancellation", following Data points should be updated for every lot in that consignment whose status is "In Packaging":

  1. Cancelled By - Seller
  2. Cancelled On - TimeStamp
  3. Reason of Cancellation - As given by seller
  4. Cancelled Order Item ID
  5. No. of Lots Cancelled
  6. Status at the time of cancellation

Also, hide that Consignment from this page.

Impact of Cancellation:

a. Send Push Notification and SMS to the buyer about cancellation. Now, check if the no. of lots cancelled is equal to the no. of lots in that order, (In this case there can be multiple orders in one consignment, therefore check for every unique order)

  • If yes, then send Push Notification "%seller_name% has cancelled your order - %skorderID%"
  • If No, then send then the push notifications "X lots have been cancelled by the Seller - %seller_name% from your order - %skorderid%     

b. Send SMS to Buyer about cancellation. Use above condition here as well

  • If yes, then send Push Notification "Dear Buyer, Seller - %seller_name% has cancelled your order - %skorderID%. Check 'My Orders' Section in the app for updates."
  • If No, then send then the push notifications "Dear Buyer, %X% lots have been cancelled by the Seller - %seller_name% from your order - %skorderid%. Check 'My Orders' Section in the app for updates"

Seller must select the no. of lots to cancel from the dropdown. The Dropdown will contain values in lots of the variant. Minimum Value being '1' & Max. value being the (Lots ordered - Lots Cancelled).

Seller will also select the reason of Cancellation. (List of Reasons will be provided soon)

  • If yes, then send Push Notification "%seller_name% has cancelled your order - %skorderID%"
  • If No, then send then the push notifications "X lots have been cancelled by the Seller - %seller_name% from your order - %skorderid%     

b. Send SMS to Buyer about cancellation. Use above condition here as well

  • If yes, then send Push Notification "Dear Buyer, Seller - %seller_name% has cancelled your order - %skorderID%. Check 'My Orders' Section in the app for updates."
  • If No, then send then the push notifications "Dear Buyer, %X% lots have been cancelled by the Seller - %seller_name% from your order - %skorderid%. Check 'My Orders' Section in the app for updates"

14Cancellation by buyer/seller info

There can be two Cancelled statuses for each variant (one for buyer and one for seller), therefore group the cancellations by Buyer and Seller separately.

On hovering on this status, it will the show the reason of cancellation in a tooltip. As we will on club and show the cancellations by Buyers and Sellers there could be more than one reason for cancellation for each. Hence, in this case show multiple reasons on hovering.

If all the Lots of any variant gets cancelled, then the whole variant block will become translucent to 50% and “options” button and “move to packaging: button will be disabled. (Although, the cancellation tooltip should still work)

New


Checks for Cancellation

Validation for Cancelling and Hiding Product Row, On Cancellation at Lot Level,

Check whether all the other variants in that order are marked Cancelled or not (i.e. sum of the remaining qty of all variants in that order = 0),

  • If Yes, then update cancellation mark at the Order Level as well [Status = 5, Cancelled By, Cancelled On] and remove the consignment ID and reset the consignment ID for that buyer and Seller.
  • If No, then check if all the other variants in that consignment having status "In Packaging" are marked Cancelled or Not.(i.e. sum of the remaining qty of all variants in that consignment= 0)
    • If Yes, then hide/remove that consignment row from the page and reset Consignment id between that Buyer and Seller..
    • If No, then check if all the other variants in that product which have status "In Packaging" have been cancelled or not:
      • If Yes, then hide the product entry
      • If No, then mark that variant as cancelled and reduce the translucency of the variant block to 50% and “options” button will be disabled. (Although, the cancellation tooltip should still work).

Validation for Consignment ID resetting, On "Cancellation" or "Move back to In Process", run a check,

If for the given consignment ID are there any items left which are mapped with that consignment.,

  • If Yes, then do nothing
  • If No, then reset the Consignment ID.(i.e when any other lot is being moved to packaging, then new Consignment ID will be generated.)
