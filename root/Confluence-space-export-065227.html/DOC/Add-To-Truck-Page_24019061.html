Documentation : Add To Truck Page

Created by Siddharth Rastogi, last modified on Mar 26, 2018

Overview

Here the users will make final decisions for purchasing a product. Lot Size, Carton Size, Size Sets, MOQ, variant color etc.

On landing on this page, app needs to check the various product configurations to populate the page accordingly.  


This Page will have majorly 4 Variations based on the product Configurations.

Case 1: Lot Wise (Include Color = Yes)

Case 2: Lot Wise (Include Color = No)

Case 3: Carton Fixed (Include Color = Yes)

Case 4: Carton Fixed (Include Color = No)

Case 5: Carton Felxible

Case 6: Lot Wise (Include Color = 0/Null)



Basic Page Details

Data Points

S.NoData Points / ButtonsLogic
1Brand Name and CounterName of Brand and Counter assigned to the Variant
2Sk Price and MRPFinal SK Price. Show MRP if MRP > 0
3MOQ/Carton

"MOQ", if Sell Type = Lot

"Carton", if Sell Type = Carton

Appended by value of "MOQ Pairs"

4Lot Size

Show this in brackets appended to the MOQ/Carton text. 

"(1 Lot = X Pairs)" ; where X = Lot Size Value 

5Minimum Buying Details Text

This Text line will appear below the "MOQ Pairs" Value. This text will be dynamic and the logic will be defined at API Level.

The logic for the Text has been explained in detail below.

6Info Icon

On click of this icon, open a tooltip with following information:

  • Seller Name
  • Seller MOQ
  • Expected Delivery

7Variant Block

By Default, there will be only one variant block populated with first variant color image (in case multiple variant present). It will have following actionable points:

  • On Selection of No. of Pairs, show "SK Price" x "Pairs Selected" = "Total amount". In case there is Extra Charge for selected Size Set, then append and show the Extra charge amount.
  • Color Name and "Change Color" Button. On click, open a dialog from below listing  
8Choose Size Set

On click, open dropdown with the list of size sets for the product.

  • Validate for Limited Stock Value. In case a limited Stock value is active for a particular Size Set and the Stock lot value is '0', in that case do not show that Size Set.
9No. of Pairs

On click, open list of numbers. The logic of numbers is defined based on the cases mentioned above.

10Price , pairs and Total

This will be visible only once the 'no. of pairs' selection has been made. It will show SK Price(including extra sk price if any), multiplication symbol, Value of No. of pairs selected, Total Amount.

In case the selected Size Set has extra charge attached to it, then show the extra charge value next to the Total Amount. 

Also, in case the total no.of pairs chosen is such that it falls under bulk pricing, then update the SK Price both in variant block and in top section. Animate the SK Price Value after it has changed (Use shake animation). Also, show toast "Bulk price has been applied"

11Color Name 

Show the color name of the selected variant

12Change Color button

In case the no. of variants in the product is only one, then keep this button hidden.

In case the no. of variants in the product are more than one, show this button.

On click, open a bottom dialog box, displaying all the available colors. On click of any color, the dialog will close & its respective variant block will be created and displayed on the page.

13Remove Button

This button will be hidden/disabled if only one variant block is present at any given time.

On click, remove the Variant block

14Add Color Button

On click, open the variant color selection dialog from bottom. Same as "Change Color" button.  

Keep this button, in case the product has only one variant and one size set.

15Fulfill Pairs Box This box and its text will be dynamic. This box will display whenever the minimum pairs requirement is not fulfilling. The dynamic texts will change based on the product configuration and cases defined above.
16Total Amount The Total Amount after adding the totals of all variant blocks
17Total PairsThe total pairs count after adding the total pairs of all variant blocks. 
18Add to Truck Button

By default, this button will be in a Disabled state. 

This button will be enabled only when the minimum pairs requirement has been fulfilled.

This will also depend on the cases based on the product configuration.

On click of the Add to Truck button, items will get added to the cart and app will open the Product listing page.

Add to Truck Animation Idea

On click of 'Add to Truck' button, show a circular loading animation inside the button area. While the loading is in progress, An image of a Carton will appear from behind the ATT Button and travel upto the truck icon while zooming out.


Validations


Logic for Buying Details Text


Case 1: Sell Type = Lot, Include Color = No,

"You can buy minimum of X pairs of single color" ; where X = MOQ Pairs

Case 2: Sell Type = Lot, Include Color = Yes, (more than one variant)

"You can buy minimum of X pairs including multiple colors" ; where X = MOQ Pairs 

Case 3: Sell Type = Lot, Include Color = Yes, (one variant)

"You can buy minimum of X pairs of single color" ; where X = MOQ Pairs 

Case 4: Sell Type = Carton, Carton type = Fixed, Include Color = Yes

"You can buy minimum of X pairs of fixed colors and Size set" ; where X = MOQ Pairs 

Case 5: Sell Type = Carton, Carton type = Fixed, Include Color = No

"You can buy minimum of X pairs of single color" ; where X = MOQ Pairs 

Case 6: Sell Type = Carton, Carton type = Flexible (one variant)

"You can buy minimum of X pairs of single color"; where X = MOQ Pairs

Case 7: Sell Type = Carton, Carton type = Flexible (more than one variant)

"You can buy minimum of X pairs including multiple colors" ; where X = MOQ Pairs

Case 8: Sell Type = Lot, and MOQ = Lot Size

"You can buy minimum of X Pairs of single color"; where X = MOQ Pairs
