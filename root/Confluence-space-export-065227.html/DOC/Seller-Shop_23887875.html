Documentation : Seller Shop

Created by Siddharth Rastogi, last modified on Mar 26, 2018

Overview

A Seller shop is basically consists two elements:

  1. Seller Shop
  2. Seller Details Popup

This will allow buyers to view information about sellers to make buying decisions such as Seller MOQ, Cancellation rates etc. and will allow users to browse specific sellers' products.

1. Seller Shop Products

Data Points

S.NoData Points / ButtonLogic
1Background ImageThis will be a dynamic image. URL passed by API
2

Seller Name

Company Name of the seller registered
3Profile TypeProfile type selected at the time of registration

CityName of city with whcib
5Seller MOQ

Show Value, if Seller MOQ > 0

Show "No Seller MOQ", if Seller MOQ = 0 


Total ProductsShow text "Total X Products" ; where X is the value of total variants uploaded by the Seller.

Seller Details On click, open a popup showcasing details of the Seller. Explained in below.
6Showing Brands

Case 1 - If one brand is listed, then hide this whole section and start with the product Listing.

Case 2 - If more than one brand is listed, then show this section.

By Default, all the brands must be in selected state. User can click on a brand to deselect it. The Product Listings should be filtered accordingly. 

In case only one Brand is in selected state and user tries to deselect it, do not deselect that brand and show toast "Atleast one brand must be selected"

This section will be horizontally scrollable.

7Product ListingThe order of the products listed must be dynamic and handled through APIs
8FiltersWe will use new filters in subsequent release. For initial release we will maintain our existing filter system.

Other Notes

  1. Load minimum of 20 Variants in one page
  2. We will use Parallax effect for the upper container.
  3. The Filters will stick to the bottom.




2. Seller Details PopUp

This popup will open on click of "Seller Details" button.

Data Points and Logic

S.NoData points / ButtonsLogic
1Analytics

This visibility of this section needs to be dynamic such that we can hide/show this section based on a key value given by API.

Logic for when to show and hide these sections is yet to be decided.

Cancellation Rate

(Total pairs cancelled by Seller / Total no. of pairs ordered) * 100

Return Rate

(Total pairs returned by buyers / Total no. of pairs delivered) * 100

2Seller Details

Concern Person - Name of company representative 

Location - City, State

Avg. Dispatch Time - Average of the Dispatch time of all products uploaded by seller

Categories Sold - This will show max of 3 categories. Each category will be a combination of Main Category and subcategory. The categories shown here will be based on the maximum no. of products of the seller in a particular category combination. Do not break the line if the text is exceeding the available space.
