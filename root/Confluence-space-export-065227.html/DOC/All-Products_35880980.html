Documentation : All Products

Created by Siddharth Rastogi, last modified on Apr 19, 2018

Overview

This is an internal module for the Admin Panel where the team can view all the Products that have been Approved via the Product Review Process. 

Scope of work:

#1 Allow admin to apply filter, sort & search from the list of products.
#2 Admin could Activate/Deactivate Variants
#3 Show admin all the information about the product

Three Parts:

  1. Main Listing Page
  2. Image slider popup 
  3.  Product Details popup

*Note for Dev - Keep a special focus on the Search & Filters options and how they work along with pagination.

  • Download CSV (Not currently.)

Zeplin URL - zpl.io/VOB4x4k


The Page title will be "All Products" followed by the total product count on the App. (Logic - Count of Unique product ids where "ApprovedOn>0")

Below the Sorting/Filters box, Show the following texts (as shown in image below):

Total Variants: (Logic - Count of Unique variant ids where "ApprovedOn>0")

Active Variants: (Logic - Count of Unique variant ids where "ApprovedOn>0" AND "isAvailable=1")

Inactive Variants: (Logic - Count of Unique product ids where "ApprovedOn>0" AND "is Available=0")

Sorting : By default Descending order based on ApprovedOn (approved Date).

Filters :  Yet to discuss....

Note - The above values will change as per the filters/search queries applied. In case no filters or search query is there it will simply show the total values. 

Data Points and Logics (Main Listing page)

#Data Points / ButtonsLogic
1Product Name 

Name of the Brand. Appended with the brand counter. 

[Brand Name Character Limit - 20]

2Seller Name 

Name of the Seller (using Seller ID of the product)

[Character Limit - 30]

3Sell type 

Check field "sellsType" in products Table:

  • If sellsType=0, show text "Lot"
  • If sellsType=1, show text "Lot"
  • If sellsType= 2, show text "Carton"
4Include Color

Check field "multipleColors" in products Table:

  • If multipleColors=1, show text "Yes"
  • If multipleColors=0, show text "No"
5Lots SizeData from field "lotSize" in Products Table
6Product MOQData from field "moqPairs" in Products Table
7Seller MOQData from field "sellerMOQ" in user_master Table
8MRP Data from field "mrp" in Products Table
9SK PriceData from field "skPrice" in Products Table
10Category

Show the category text in the following format:

'Main Category'>'Sub-Category'>'Style'

[Total Character Limit - 35]

11L/S

Data from field "isLeather",

  • If 0, show text "Synthetic"
  • If 1, show text "Leather"
12UpperData from field "matUpper" in products Table
13LiningData from field "matLining" in products Table
14SoleData from field "soleTypeId" in products Table. Fetch the name of the sole via term type relationship tables.
15Approved On Show Date of Approval. Field "approvedOn" from products Table 
16Variant Image ThumbFetch the image url from "defImage" field in variants table. On click of any variant thumbnail image, a popup shall open showing all the angles.
17Variant ColorData from field "colorId" in variantsTable. fetch the color name via colors table 
18Article No.Data from field "brandCode" in variants table
19Active/Deactive Toggle

This toggle will set the "isAvailable" field.

If Active, set value to "1"

If Deactive , set value "0"

20View Details ButtonOn click of view Details button, open a "Product Details" popup. Explained below. Refer to the bottom of the screen. (Explained further below separately)

Data Points and Logics (Product Details Popup)

#Data Points & ButtonsLogic
1Base PricingContains MRP, Seller Price (upto two decimal places, SK Price) from products table
2Bulk Pricing

This section will be visible only if following value of the product is present in the "product_bulkprice" table:

a. 'MOQ' as MOQ

b. 'price' as Seller Price (upto two decimal places)

c. 'skPrice' as SK Price

This section must remain hidden in case no value are there in "product_bulkprice" table.

3WarrantyThis texyt appears on right of Base Pricing Section. Fetch values from the field "warrenty" in products table
4Dispatch TimeFetch data from field "minLeadTime" in products table.
5Size Sets and Extra Charge (if any)

This section appears on bottom of popup. List down all the size sets mapped with the product in table "product_setsizes".  Find the Size IDs in this table and fetch its value via Size_master Table.

Also fetch the "skPrice" as Extra Charge and display next to the respective Size Sets.
