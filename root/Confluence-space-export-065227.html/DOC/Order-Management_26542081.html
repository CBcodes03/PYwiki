Documentation : Order Management

Created by Siddharth Rastogi, last modified on Apr 03, 2018

Overview

This module will allow sellers to manage Orders right from receiving a new order to consignments getting delivered. This module incorporates the structural and conceptual changes discussed for the marketplace approach and implemented in the Product Management Module. The aim is to make the order processing as easy and swift as possible.

This Module can be divided into following parts:

  1. New Orders (old)
  2. Buyer Info Popup
  3. In Process
  4. Expected Dispatches (No doing in this release)
  5. Packaging - In Packaging (old)
  6. Packaging - Initiated Pickup
  7. Dispatched - In Transit, Delivered, RTO

User Flow

This process will start as sellers receive New Orders on their Board. If the Order has been confirmed, then the orders will directly show in “New Orders”. Sellers can move the orders from "New Orders" Screen to "In Process Screen. Here Sellers can cancel SKU’s at Lot level or Move items to Packaging. Sellers can also keep a check on cancellations done by the buyer. Once, moved to packaging, the items will show up in “Packaging” Screen under the “In Packaging” tab. Seller can move items back to “In Process” from “Packaging” Screen. Now Seller will initiate Pickup. Once initiated, the Consignment will show up in the “Initiated Pickup” Tab. Here Seller can download documents related to the invoice or re-request pickup.

Once the Logistic partner’s API updates the status, this consignment will start showing in “Dispatched” tab. Here the consignments will be trackable with location. Once Delivered, it will start to appear in “Delivered” Tab. If RTO, it will show in Returns Tab.


For Next release:

New Orders: Orders will start showing in "New Orders" Screen if the buyers are verified by Admin. The Confirmation of orders will be done by the sellers from "New Orders" Screen. On Confirmation, move those orders to In Process.
