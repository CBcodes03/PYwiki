Documentation : Old Consignments

Created by Siddharth Rastogi on Jun 11, 2018

Overview 

The Goal is to eliminate all the functions where "ods" is being used and fix the logics if required on the basis of the new Structure.

1. Check Logics of the following data points (for ods dependencies)

  1. Carton Count
  2. No of SKUs
  3. No of Pairs
  4. Consignment Value
  5. Delay Data
  6. Amount Payable

2. CTA Logics

  1. When Status = "Initiated Pickup", show CTA "Mark Dispatch"
    1. On Click, set TimeStamp in "actDispatchDate"
  2. When Status = "Dispatched", show CTA "Mark NDR" and "Mark Delivered"
    1. On click of "Mark NDR", set TimeStamp in "ndrDate"
    2. On click of "Mark Delivered", set "actDeliveryDate" and add the value of SUM (od.remainingSetCount*od.setCount) where cm.Consignmentid=od.consignmentid in "o.completedPairs" 
  3. When Status = "NDR", show CTA "Mar RTO" & "Mark Delivered"
    1. On click of "Mark RTO", set TimeStamp in "rtoDate" and add the value of SUM (od.remainingSetCount*od.setCount) where cm.Consignmentid=od.consignmentid in "o.completedPairs"
    2. On click of "Mark Delivered", set "actDeliveryDate" and add the value of SUM (od.remainingSetCount*od.setCount) where cm.Consignmentid=od.consignmentid in "o.completedPairs"
  4. When Status = "RTO", show CTA "Mark Warehouse Delivered"
    1. On click, set currentTimeStamp in "actDeliveryDate"  
  5. When Status = "Delivered" or "Warehouse Delivered", Do not show any CTA

3. Check ods Logic in the Status Filters, Filters, Sorting, Search
