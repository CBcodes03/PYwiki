Documentation : My Truck

Created by Siddharth Rastogi, last modified on May 07, 2018

Overview

When a user adds products to the Truck, they land up on this page grouped by the seller names. Users can reach this page by clicking on the Truck icon present on the Toolbar. Few new concepts are to be introduced here.

a. Unlike existing app, now when a user adds multiple products from different sellers, for every seller a separate order id will be generated. 

b. Users cannot checkout a seller if its Seller MOQ requirement is not fulfilled.  

c. Users can Visit the sellers shop, edit Order or Remove Seller Order 

d. User can check multiple sellers and place multiple orders from his Cart/Truck in one go.

On click of Remove, Remove the order entry for that specific seller from the list (Delete the entry from database).

7Info text Box

This is a red info box that appears above the Order Value text. This box will display texts to the user in case there is a Seller MOQ or Product MOQ violations. This box will appear only in two cases:

Case 1: If the no. of pairs of any product in an order becomes less than the Product MOQ. This could be because any product became inactive (isAvailable = 0) or is StockLot of any variant become 0 or brand is Deactivated. etc..

  • Show text: "EDIT ORDER: X Product's MOQ does not match the order qty" ; where X = Count pf products which have this violation.

Case 2: If the no. of pairs of any order is less than the sellerMOQ. 

  • Show text: "Add X more pairs to complete Seller MOQ" ; where X is the difference of pairs between pairs ordered and sellerMOQ

Case 3: In all other cases do not show this box. 

8Checkbox

When the whole page loads, for every order check whether the no. of pairs are equal to or more than the Seller MOQ of that order.

  • If Yes, set the checkbox in checked state automatically. (User can click on it and un-check the checkbox. This will affect the "multiple order checkout" button as explained below.) 
  • If No, set the checkbox in disabled state. (In this case if user clicks on the checkbox, show toast "Add X Pairs to place the order") 
9Order Value and Pairs

This will show the sum value of all products in that order [Sum of (SK Price x Pairs)]. This will be appended by the count of total pairs ordered. The color of the text will change according to following condition:

  • If Pairs Ordered < Seller MOQ, set the color as Red #FF6565
  • If the Pairs Ordered ≥ Seller MOQ, set the color as Green #00C29C

Add Products Button

This button will have three possible variants based on the below conditions:

  1. Seller Checkout Button This button will be visible when there is no product moq violation or seller moq violation. On click, make necessary API calls and open the Checkout Page. Button Color #00C29C
  2. Edit Order Button This button will be visible when product MOQ violation is there in an order. On click, open the Edit Seller Order Screen. Button Color #FF6565
  3. Add Product Button This button will be visible only when the there are no product MOQ violations but there is seller MOQ violation. On click, open the Seller Shop Page. BUtton Color #FF6565
11Multiple Orders Checkout Button

This is a floating button that will appear at the bottom of the screen. It will be visible if more than one checkbox is in selected state. 

  • If more than one checkbox is in Selected state, then this button must be visible.
  • If only one or no checkbox is in Selected state, then this button must be in hidden state.

Use animation to slide up and slide down the button according to the above conditions.

The data points on the button will be:

  1. text "Multiple Orders Checkout"
  2. Total Order Amount  = Sum of the all the checkboxed orders' amounts
  3. Total Pairs = Sum of all the checkboxed Orders' pairs
  4. Text "X Orders"; where X = count of Orders with checkbox in selected state

On click, call the necessary APIs and open the Checkout Page
