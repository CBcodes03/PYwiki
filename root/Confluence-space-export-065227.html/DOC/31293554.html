Documentation : Order Details > Items List

Created by Siddharth Rastogi, last modified on Jun 01, 2018

Overview

Here users will be able to see the statuses of all the order items. This page will list down all the variants grouped by products.

This page has two major functions:

  1. Tracking items' status
  2. Cancellation

Variant Statuses


Every Variant will show statuses below it.

There can be maximum of 3 different statuses of one variant:

  • Movement Status
  • Cancelled by Buyer
  • Cancelled by Seller 

Each status will have following attributes and they will be dynamic:

  • Text 
  • Text Color
  • Bg Color
  • Info Icon - Will appear only with cancelled Statuses

2. Cancellation

A Buyer can cancel the whole order. (Variant Cancellations will be introduced in later releases)

Cancel Order

This will be available inside the option menu on the Toolbar. 

In case all the items of that order are already cancelled or all the items have status Pickup Initiated and above, then keep this option Disabled.

  1. If Order status = Placed, then cancel order button enabled.
  2. If order status = Cancelled or Complete, then "cancel order" button disabled
  3. If order status = confirmed, "Cancel order button" enabled. On click, check
    1. If all items of that order have (order_stage), then show popup "You order cannot be cancelled as all the items have been initiated for pickup". (handling by API)
    2. If all items of that order are either fully cancelled OR does not have status (In Packaging, In Process, New Order), then show popup "Your order cannot be cancelled as all of the items have either been initiated for pickup or already cancelled." (handling by API)
    3. Else, show Bottom sheet for entering reason of cancellation


On click, Open a bottom sheet which will ask for the reason of cancellation. The list of reasons will be dynamic i.e Given by APIs.

On click, of "Cancel Order", validate that atleast one reason should be selected. If 'other reason' option is selected, then validate that the Edit box must not be empty.

 

On Click, of Cancel Order, check if all the variants are eligible to be marked Cancelled (i.e all variants have any of the following statuses "Confirmation Pending", "In Process", "In Packaging"):

  1. If Yes, open popup: "All items of this order will get cancelled. Are you sure you want to cancel ?" [ Options YES | NO ]. 
  2. If No, open Popup: "X items of this order will get cancelled. Are you sure you want to cancel ?" [ Options YES | NO ]

On click of  'YES', the following actions should be performed.

  • For case 1 above, Update the cancelled mark at Order Level [Status = 5, Cancelled On, Cancelled By, Remaining Pairs=0] and also update at order details level. Also insert respective cancellation log entries. 
  • For case 2 above, Update status of all variants having any one of the status: "Confirmation pending", "In Process", "In Packaging" and mark them cancelled with the respective reason.

Web Serivice Tasks on Cancellation

On Cancelling the Order, the following tasks should happen in Database.

Pending

Alternative Flow:


  1. Orders Listing Api will give the data for no. of order items elligible for cancellation. (order_stage !=3 and remainingSetCount>0) Suppose, itemsCancellable = X
  2. In Orders Details Page, on toolbar,
    1. if the itemsCancellable = 0, then show Cancel Order Button Disabled. On click, show popup "This order cannot be cancelled as all the items have either been initiated for pickup or already cancelled"
    2. If the itemsCancellable > 0, then show Cancel Order Button as enabled. On click,
      1. If itemsCancellable = no. of order items, show popup : "All items of this order will get cancelled. Are you sure you want to cancel ?" [ Options YES | NO ]. 
      2. If itemsCancellable != no. of Order items, show popup: "X items of this order will get cancelled. Are you sure you want to cancel ?" [ Options YES | NO ]
    3. On Click of "Yes" in either of the above case, open Bottom Sheet.
    4. After User selects the reasons, when he clicks on final "Cancel Order" button, on success, show message "Items Cancelled Successfully" and refresh the page to reflect the changes made.
