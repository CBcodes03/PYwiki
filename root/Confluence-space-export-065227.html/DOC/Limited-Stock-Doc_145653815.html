Documentation : Limited Stock Doc

Created by Siddharth Rastogi, last modified on May 29, 2018

Overview

This is basically the part of product uploading where seller can manage and update the stock lots for its products. 

This button will be enabled only when at least one size set and one color has been chosen.

There are two types of stock lots:

  1. For all cases except Fixed Carton Acc:Yes
  2. For Fixed Carton Acc:Yes

On click of the button, a popup will open (based on the above type) where seller will be able to input Limited Stock. The popup will Display the Variant Color and its listed sizes. For each size set there will be 

  • Active/Deactive toggle
  • Number Input Edit Box (Number input only. Max digits = 3)
  • For Type 1 - fetch details from variant_size_stock table. 
  • For Type 2 - fetch the details from the product table. (limitedCartonStatus, limitedCarton)

By Default (first time), all the toggles will be in Deactivated state.

The Edit box will be in a locked state if toggle = Deactive, and in open state if toggle = active. A user will have to Activate the toggle in order to enter the stock lot value.

Show the Edit Box prefilled with value of stock lot if already set in Db. 

Addition: Add a text on top "Warning: If the Limited Stock is Active and Remaining Stock is '0', then it will not be visible to the buyer.


On click  "Continue" Button,

  • In Product Uploading Page > Type 1 - Update the Ui for required variant blocks with the values of Stock lot.  (Sum of stock lots for each variant) from 'variant_size_stock' 

Visibility of Stock Lot in Uploading Page.

When a user sets any positive values and having Activated Stock Lot , then they will appear in respective variant blocks.

For Type 1 - The Total Stock Left will show the value of SUM of Activated stock lots for each color/variant.  

For Type 2 - The Total Stock Left will show the value of Carton Lots if Stock Lot is activated. The Value will be Sum of value selected for each size set for each color. (will explain elaborately)

Resetting Values

  1. When changing the Sell type to (Sell Type = Carton, Carton type = Fixed, multiple Colors = Yes) or vice cersa, Check if any Stock lot is Activated for any size set (Type 1 or Type 2). If yes, then show warning popup, "By changing the this option, your stock lots will be reset. Are you sure you want to continue ?"  On click of Yes, reset the values in Front-end. Now, when user clicks on Save Draft or Submit for review, Delete the old Stock lot Values and update the new values in Db.
  2. When deselecting a Size Set, check if there is an entry of that size set for all variant of the selected product. Update the Stock Lot Value on variant block and popup in front end. Now when user clicks on Save Draft or Submit for Review, Delete/update the Stock lot value in Db according to the type 1 or type 2.
  3. When Changing the color of a variant, check if any stock lot is activated for that variant(type 1 or Type 2), then update the limited stock pop to show the change in color.  Do not do any changes in Db for this.
  4. When removing a variant block, Remove the Variant entry from display in Limited Stock popup. On Click of Save Draft or Submit for Review, Delete the Stock lot entry for that variant and Sizeids (if Type 1) and update the caluclation (if Type 2).
  5. On Changing Lot or Main Category, Check if there exists Active stock lot (Type 1 Or Type 2), if yes then add a text in the existing warning popup "Also, the limited Stock lot values will be reset." 


This Button will also appear in the Product Edit Mode and have the same functionality.




For Type 1

Case 1 - On opening the popup

  1. Fetch the values from variant_size_stock table, for variantids and sizeids of that product. And show these entries as "Activated"

Case 2 - On Click of "Continue" Button in the popup

  1. In Frontend - in the variant block, show the sum of all the activated stock lots for size sets
  2. In frontend - set variables of active/inactive for any changes made by the seller. 

Case 3 - On click of "Save Draft" or "Submit for Review"

  1. Delete the old stock lot entries 
  2. Update the new stock lot entries
