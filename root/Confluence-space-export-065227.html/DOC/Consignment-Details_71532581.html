Documentation : Consignment Details

Created by Siddharth Rastogi, last modified on Apr 26, 2018

 Overview

This page will open on click of the "View Details" Button or "Images area" from the Track Consignment Page. This page will show all the details regarding the consignment to the buyer. The details will be majorly the following:

  1. Tracking Details
  2. Consignment Values
  3. Items List

Data Points & Logic

#Data Points & ButtonsLogicStatus
1Consignment IDThis appears right at the top of the screen Existing
2Date 

This date appears right below the Consignment ID. The date will follow the same logic in the Track Consignments Screen. The Date label and value shall change depending on the tab under which that consignment is appearing.

  1. In Transit Tab: Show text "Dispatched" followed by the date cm.actDispatchDate
  2. Delivered Tab: Show text "Delivered" followed by the date cm.actDeliveryDate
  3. RTO Tab: Show text "RTO On" followed by the date cm.rtoDate
New
3Seller Name

Show label "Seller Name". Below the Label show the name of the Seller.

If the value of Seller name exceeds the width of the screen then show "..."

New
4Status & Remarks

The status has exactly the same logic as given in the Track Consignments Screen. 

Show Main Status, Logistics Remark and the info icon.

On Click of info icon, open a tooltip showing the full logistics Remark Text.

New
5Tracking No.This is the value of Waybill No. generated for that consignment.Existing
6Track Details ButtonThis button appear to the right of Tracking No. On click, open webview in the app and open the Logistic Partners' Tracking URL (saved in Database) and append the Waybill no. to that URL so that inside the webview, the specific consignments' details get opened.Existing
7Consignment Details 

This Section will display all the values and amounts related to that Consignment.

  1. "Cartons" - Show the count of Cartons 
  2. "Total Pairs" - Show the total pair count of that consignment
  3. "Amount" - Shows the Consignment Value.
  4. "Amount Payable" - Shows the value (cm.consignmentValue - cm.advancePaid)
Existing
8Get Invoice on Mail

On click, open a popup. 

Show this edit box empty if, the username has value in format: "mobilenumber"@shoekonnect.com

else, show this edit box prefilled with the username value. 

On click of "Send Invoice" Button, validate the email format and empty values.

On failing validation, show toast "Please enter valid email Id"

On passing validation, check whether the given email id exists against any other user in the database.

  • If Not, save the value as the user's 'Username' and trigger the invoice email on that email id 
  • If Yes, then show toast "This email Id belongs to another user" 
Existing
9Items in Consignment

This Section appears below the consignment Details Section. This lists down all the order items mapped with that consignment.

Data points for each item entry:

  • Image Thumbnail
  • Product Name (Brand Name + Brand Counter)
  • Variant Color 
  • Size Set
  • Price (skPrice + extSkPrice)
  • Pairs 
  • Total Amount (Price x Pairs)
