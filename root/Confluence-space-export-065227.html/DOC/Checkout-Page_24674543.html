Documentation : Checkout Page

Created by Siddharth Rastogi, last modified on May 26, 2018

Overview

Objective: To simplify and minimize the existing checkout process into 2 to 3 Steps.

Structure

This will be a slightly lengthy page but will pack all the pre-order information at one place.

The main sections here will be:

  1. Shipping
  2. Payment
  3. Verificatiion
  4. Bill Summary
  5. Apply Coupon

Have explained each section in detail below.

1. Shipping

This section will display the Delivery address chosen by the user for delivery of the consignments. This will also allow the user to change or add new addresses.

On the right we have two views of this section in following conditions:

a. No address provided.

b. Address already provided

Data Points and Logic

S.NoData Points / ButtonsLogics
1Shipping Address TitleThis is the title on the Top "Shipping Address"
2Address Area

When no address is provided, show text

  • "Please provide address for delivery"

When address is already provided, show address in given format

  • Company Name (In bold)
  • Representative Name (if provided)
  • Street Address
  • City - Pincode
  • Phone No. 
  • Optional Phone No. (if provided)
3Delivery Available info text

Every time the page loads, app will check if a selected address exists. 

If selected address exists, check for Pincode Servicibility of the selected address:

  • If pincode serviciable, show check icon and text " Delivery available at above address" in theme green color
  • If pincode not serviciable, show cross icon and text "Delivery service not available at above address" in red color.

If no selected address exist, then do not show this line at all.

3Add New AddressOn click, open the "Add New Address" Page. Described about it more below.
4Change Address

This button will be visible only when an address is already selected. On click, open the My Address List Page. Described about it more below.-

3. Verification

Here users will give any one form of verification. 

If they choose GST option, they shall provide the GST No. 

If they choose "i do not have GST", they shall give any one verification image.

This however is not a mandatory step. If a user selects any option and does not provide the verification document or gst no. we shall not stop hime from placing the order. 

Data Points & Logics

S.No Data Points / ButtonsLogics
1GST Option

This option will remain selected by default. When this option is selected, the below area must show a Edit box for inputting a GST No. (Max input limit: 15 characters)

Show information below the Edit box "To get GST invoice and tax benefits, please provide your GST Number above."

If the GST No. has been approved, then keep this edit box in a locked state.

Case 1:  GST Edit box will be on open state if, isGSTVerified=0 (and show pre-filled value if any)

Case 2: GST Edit box will be locked if, isGSTVerified=1 (and show pre-filled value if any)


  • GST input format validation: 2 Digits Followed by 5 English Characters followed by 4 digits followed by 1 character followed by 1 digit followed by 1 character followed by 1 digit 
2Other Verification Option

On selection of this option, the below area will change (as shown in image).

Document Dropdown Options:

  • Bill Book
  • Visiting Card
  • Shop Photo

Image Upload button: On click, open the image selection tool and upload the image. On successful upload, show a thumbnail preview in the image box. If user clicks on the Image Upload button without selecting any dropdown option, then do not open image selection tool and show toast "Please select one document option" 

If the verification document is approved by Admin, then keep both the Dropdown and Image upload button in locked state.

CASE 1 : Keep the Dropdown and Image Upload Button in active state, if orderVerifyDone=0 (Show pre-filled data if any)

CASE 2: Keep the Dropdown and Image Upload Button In Locked State, if orderVerifyDone = 1 (Show pre-filled data if any)

CASE 3: Keep the "I dont have GST" option disabled if, isGSTVerified=1 (so user will not be able to open doc verification box) 

On click of why option, open a popup displaying the reason for verification. Use the existing popup design and content.

3Verification Disclaimer box

This text box will appear below the verification section only in following case:

  • Show text box if, um.isGSTVerified = 0 AND um.orderVerifyDone = 0
  • Hide this box if, um.isGSTVerified = 1 OR um.orderVerifyDone = 1

Keep the message dynamic via API.

4. Bill Summary

Here the complete bill summary will be listed. 

Data Points and Logic

S.NoData Points Logic
1Total SellersShow the no. of sellers for whom the user is checking out.
2Total PairsSum of pairs of all orders
3Subtotal This is the sum of subtotal value of all orders [Sum of (SK price x Pairs)]
4Coupon DiscountThis row will appear only when user applies any valid coupon. Show the value with negative sign.
5Shipping Charge

Every time this page is loaded, run API to calculate the Shipping charge for all the orders on separate basis.

i.e If Order 1 has 24 pairs and Order 2 has 48 pairs, then shipping charge will be calculated for each order separately and shown here as a sum. ( Instead of calculating shipping charge for 48+24 = 72 pairs )

Note - Shipping charges will include the Shipping tax values.

6Advance Discount

This row will appear only if the payment option selected is "100% Advance".

The value to be deducted here should be dynamic as the discount percentage will be kept dynamic as well.

7Grand Total

This is the sum of all the values appearing above:

Subtotal - Coupon Discount + Shipping Charge - Advance Discount = Grand Total

820% Advance Amount Box

This box will be visible below the summary section only in case the payment option selected is "20% Advance | 80% COD".

This will show the breakup of 20% advance and 80% COD and the value payable in the next step.



5. Apply Coupon

Here users can apply a coupon code and avail discounts from ShoeKonnect.

Data Points & Logic

S.NoData Points / ButtonLogic
1Coupon Code Edit BoxUser will enter inputs here (Keep Keypad in Capital Chase characters only)
2APPLY/CLEAR Button

When user enters a value in edit box, he will click on the "APPLY" button. This will check for validity of the Coupon Code and change the button to "CLEAR". While the button is "CLEAR", user cannot manually edit the Coupon code. 

On click of "CLEAR", the edit box will be resetted and user will be able to enter a new coupon code.

3Validation text

This text will appear below the edit box. The text will be dynamic. 

On Success, "Congratulations ! Discount Coupon is applied."

On Fail, "It seems the coupon is not valid"


5. Confirm Button

This button will be the last element of this page. There will be multiple validations on click of this button.

Data Points and Logic

S.NoData Points / ButtonLogic
1Button Text

If the Payment option chosen is "Full Cash on Delivery", then the button text will be:

  • "CONFIRM ORDER" - on click, open the confirmed order page.

If the Payment option chosen is either "20% Advance" or "100% Advance" , then the button text will be:

  • "PAY NOW" - on click, open the Payment gateway
2Order Values
(total pairs)

In Case of 20% Advance,

  • Show the Amount of 20% payable amount

In all other payment options,

  • show Net amount payable
2Validations

a. Check if the Address provided is serviciable or not.

  • If not, show toast "Delivery service not available at your address"

b. Check if any payment mode is selected.

  • If no payment mode is selected, show toast "Please select any one payment mode.

c. If GST Verification option is selected, check if any value input in the edit box.

  • If the Edit box is left blank, let the user continue with placing the order 
  • if some values have been entered in Edit box, check if length of characters is 15. If Yes, proceed with order. If No, show toast "Please enter valid GST No."

d. If other verification option is selected, check

  • If any value has been selected from Dropdown options but no image is uploaded or vice versa, then on click of "Confirm" Button, let the order proceed and do not save the verification type option.
  • If no value has been selected from dropdown and no image uploaded, then let order proceed.
  • If both Dropdown value and image has been uploaded, save the both the data in user table.

e. If the Grand Total Value is less than 100. Show toast" You cannot place order for value less than Rs.100" 

3Other Points
  1. Save the Address in Order Table with proper Format.
  2. Save the GST No. at order level if Verified by Admin
  3. Save the Seller MOQ in Order Table at time of placing order.
