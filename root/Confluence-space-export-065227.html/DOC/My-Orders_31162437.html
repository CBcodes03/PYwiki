Documentation : My Orders

Created by Siddharth Rastogi, last modified on Apr 16, 2018

Overview

This page will produce a list of all the orders placed by the user and allow users to track and manage them.

This page consists of two sub-parts:

  1. Order Details > Item List
  2. Order Details > Summary


Data Points & Logic

#Data Points / ButtonLogic
1Orders ListingEach card will represent one order. The orders will be grouped by time of Order, if more than one orders were placed together. 
2Placed On DateThis is the date the order was placed by the buyer.
3Time of OrderThis is the exact time on which the order was placed. 
4Seller NameName of the seller on which order has been placed
5Status 

Each order will have one status. The following attributes of the status will be dynamic:

  • Text
  • Text Color
  • Bg Color
  • Bg Border

Listing down all the possible statuses and their conditions:

  1. Placed: [order status = 4 AND confirmedOn = 0]
    1. Text Color: #989898
    2. Bg Color: #F9F9F9
    3. Bg Border: #D3D3D3
  2. Confirmed: [order status = 4 AND confirmedOn > 0 AND atleast one variant of that order does not have any of the following statuses 'RTO','Cancelled','Delivered')
    1. Text Color: #FF9C4B
    2. Bg Color: #FFF8F2
    3. Bg Border: #F7D1B2
  3. Cancelled: [order status = 5]
    1. Text Color: #FF3636
    2. Bg Color: #FFF1F1
    3. Bg Border: #FFA6A6
  4. Complete: [all variants of that order has one of the following statuses 'RTO','Cancelled','Delivered'] 
    1. Text Color: #00C29C
    2. Bg Color: #F1FFFC
    3. Bg Border: #B0EADE
6Order AmountThe Net amount of the Order (after adjusting Shipping, discounts etc.)
7Count of Products & PairsTotal Product count and Total pairs count of that order
8SK Order IDSK Order Id Generated 
9View Items List ButtonOn click, open the Order Details page with Items List Tab selected
10View Summary ButtonOn click, open the Order Details page with Summary Tab selected
11Page Loads (pagination)While loading orders a group must not be broken due to page load. i.e the page load should decide the no. of groups to be populated in one call rather than no. of orders.
