Documentation : Initiated Pickup

Created by Siddharth Rastogi, last modified on Apr 11, 2018

Overview

This screen will list all the consignments whose pickup has been initiated and is not dispatched yet.

NOTE - We do not have to do Cancel Consignment and Move back to In Packaging in this release

Data Points and Logic

S.NoData Points /  ButtonLogicStatus
1SummaryThis will be same as the In Packaging Screen.New
2Consignment Row

The order of the consignments will be based on the latest Initiated pickup on top.

Showing Data Points:

  1. Buyer Name and City [Existing]
  2. Consignment ID [Existing]
  3. Consignment Value: Total Consignment Value of non cancelled variants (SK Price x Pairs), Total non cancelled Products in consignment, Total non cancelled pairs in consignment) [Existing]
  4. Initiated Pickup On: Date of initiating Pickup [New]
  5. Delhivery Partner - Name of logistics partner assigned for that consignment. [Existing]
  6. Documents Tab: On click of the documents tab, following options will appear: [Existing]
    - Invoice (Download)
    - Consignment Sticker (Download)
    - Print Item List (open in print preview format)
  7. Options Button: this will contain menu options “Cancel Consignment”, “Move back to In Packaging” & “Change Invoice No”.[New]
  8. Expand/Collapse Button – for expanding the view of any consignment. [New]
Change
3Cancel Consignment

On click, open a popup

“Are you sure you want to cancel the Consignment”. On cancelling the consignment, following things should happen,

  1. Update the status of consignment as cancelled.
  2. Update the cancellation status at Order Item level as well
  3. Cancelled By - Seller
  4. Cancelled On - TimeStamp
  5. Reason of Cancellation - As given by seller
  6. Cancelled Order Item ID
  7. No. of Lots Cancelled
  8. Status at the time of cancellation - "Pickup initiated"

Validation: On cancelling the Consignment, run a check on all the variants in that consignment. Since the variants can belong to multiple orders, check all the unique orders if all their variants are marked cancelled or not. (i.e sum of remaining qty of all other variants in the order = 0)

  • If yes, Remove & reset the consignment id, and update cancellation mark at order level [Status = 5, Cancelled By, Cancelled On]
  • If No, Just remove the consignment and reset it.

Cancelling Manifestation - Run manifest cancellation api and Pickup request cancellation api (if any)

New
4Move Back to In Packaging

On Click, Change the status of the consignment from initiated to "In packaging" so that it starts to appear in "In Packaging" tab

Also do the following changes:

  1. Reset all the calculated values and logistics related information from the Consignment Table.
  2. Run manifestation cancellation api (if Any)
  3. Run the Pickup request cancellation Api (if Any)

Note* This shall not merge with any existing consignment id (if present) in "In Packaging"  page. 

New
5Change Invoice No.

On Click, open a small dialog below the "documents" button, where invoice no. value will be pre-filled if it already exists. 

Seller, can enter the invoice no. or update an existing invoice no. 

Validation: Invoice no. cannot be duplicate for one Seller.
