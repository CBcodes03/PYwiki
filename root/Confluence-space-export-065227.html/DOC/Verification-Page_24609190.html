Documentation : Verification Page

Created by Siddharth Rastogi, last modified on May 05, 2018

Overview

This page is mainly accessible from the Side Drawer Menu. This page will let users give their verification details before/after placing an order. 

Page is divided into 3 Sections (View Pagers):

  1. GST
  2. Verification
  3. Bank Details


1. GST

Here user will give the GST No.

Data Points and Logic

S.NoData Points / ButtonsLogic
1GST Input Edit box

User will enter his 15 Digit GST no. here. (Max input character length is 15)

GST No. Validation: 2 Digits Followed by 5 English Characters followed by 4 digits followed by 1 character followed by 1 digit followed by 1 character followed by 1 digit

If the GST No. is Approved by Admin, this field will be in Locked state. ie. cannot be edited

2Save GST Number Button

On Click, check for validation of the GST Number and save it against the user. The number should be 15 Character Long. 

  • If GST input is not in specified format as given above, show toast "Please enter valid GST no."

If the GST no. is approved by the admin, Hide the "Save GST number" button and show the following text in its place:

"Your GST Number has been successfully verified"

2. Verification

Here user can upload any of the following document:

  1. Bill Book
  2. Visiting Card
  3. Shop Photo

Data Points and Logic

S.NoData Points / ButtonsLogic
1Document Type Dropdown

Dropdown Options:

  • Bill Book
  • Visiting Card
  • Shop Photo

If verification is approved via admin, keep the dropdown in locked state. i.e cannot be edited.

2Upload Photo Button

On Click, Open image selection tool. On selecting an image show a thumb preview of the image as well.

If verification is approved via admin, keep the image button in locked state. i.e cannot be edited.

3Why Button

On click open the popup showing reason for verification.

Use the existing popup in the current app.

3Save Verification Button

On click, check if both the Document type Value and Image URL are present.

If not, show toast" Please provide all details"

If yes, Save the details with in User table


 If the verification. is approved by the admin, Hide the "Save Verification" button and show the following text in its place:

"Your Document has been successfully verified"


3. Bank Details

Here user can update his Bank Details for refund purposes.

User will enter the following information.

  • Account Number
  • Beneficiary Name
  • Bank name
  • IFSC Code

On click "Save Bank Details", save the details and keep this page pre populated everytime its loaded.
