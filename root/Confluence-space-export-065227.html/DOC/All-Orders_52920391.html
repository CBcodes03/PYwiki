Documentation : All Orders

Created by Siddharth Rastogi, last modified on Jun 25, 2018

Overview

This is the main screen from where Admin team will manage all the orders. This page will list all the orders of verified buyers There are following main components of this page:

  1. Filters & Sorting
  2. Search 
  3. Status Filters
  4. Canceling Order Items (Keep this button hidden for first release)
  5. Download CSV (Keep this button hidden for first release)

URL to HTML  - http://design.shoekonnect.com/dash/admin/index.html


Logic of showing orders

Show all those orders that have been placed or cancelled and whose buyer has his documents verified. The orders will appear on the basis of Order Placed Date (Recent to Oldest)

BASE CONDITION: o.status = 4 or 5 AND

VERIFICATION CONDITION

(um.isGSTVerified > 0 OR um.orderVerifyDone > 0)   

Data Points & Logic

The Page Title "All Orders" will be appended by the Total Order Count (Logic: Count of orders having o.status = 4 or o.status = 5)

Explanation for Sort, Filters and Status Filters is provided below the Screen Image.

4R/N Icons

These icons appear before the Buyers & Sellers Name in Order Card. R stands for Repeat Buyer and N Stands for New Buyer.

Logic to show Repeat/New icons

For any order only one of these icons will be present at a given time. 

Showing Repeat/New icons:

Show R Icon: if the buyer has placed 2 or more orders and atleast one order has o.status=4 AND o.confirmedOn>0

Show N icon: in else condition of the above logic.

Keep orange color for R and Cyan color for N.

    2. Text Color: #1EE5A3
  3. Cancelled Order

    2. Text Color: #FC4B6C
  4. Confirmation Pending
    1. Logic: if the o.status=4 AND o.confirmedOn=0, show text "Confirmation Pending"
    2. Text Color: #26C6DA

10Lots Cancelled Status 

This appears below the SK Order ID text. There can be two statuses of this:

  1. If all the order items of that order hold true that, "od.setCount = od.remainingSetCount", then show text "No Lots Cancelled". Text Color: #1E88E5 
  2. Else, show text "X Lots Cancelled" ;where X = Sum of (od.setCount - od.remainingSetCount). Text Color: #FC4B6C
11

Products In Delay Status 

This status appears below SK Order ID text and on the right of Lots Cancelled Status. There can be two statuses of this:

  1. If all order items of that order whose (od.orderStage=1 OR od.orderStage = 2) hold the following condition true that, "(currentTimeStamp < od.expectedDispatchDate)"
    1. Then show text "No Delay". Text Color: #1E88E5
  2. If one or more than one order items of that order whose ((od.orderStage=1 OR od.orderStage = 2) AND od.remainingSetCount>0) hold the following condition true that, "(currentTimeStamp > od.expectedDispatchDate"
    1. Then show text "X Products in Delay"; where X = Count of unique products whose variants hold the condition true. Text Color: #FC4B6C
  3. If all the order items of that order have (od.orderStage=3) OR (Sum of RemainingSetCount=0) 
    1. Then do not show any text. Keep it hidden

On Click, Expand the Card Ui as shown in the design. This will populate all the variants of the order grouped by their product ids. By Default, the product row will be in expanded state showing all the variants upfront. This has two main components:

  1. Product Row (explained below)
  2. Variant Row (explained below) 
14Order Items > Product Row

Following Data points in each row:

  1. Product Name - Brand Name + BrandCounter
  2. SK Price - (od.skPrice) 
  3. Total - Sum of ((od.skPrice) * od.setCount * od.setSize) of variants having same productId
  4. Order Quantity - Sum of od.remainingSetCount of variants having same productId
  5. Cancelled - Sum of (od.setCount - od.remainingSetCount) of variants having same productId
  6. Sell Type - od.sellsType (0 - Lot, 1 - Lot, 2 - Carton)
  7. Dispatch Date - od.expectedDispatchDate
  8. B2D Delay - This value shows the delay status of each product. There can be three possible statuses : 'X Days', 'Dispatch Today', 'No Delay'
    1. If one or more than one variants of the product has (od.orderStage=1 OR od.orderStage = 2) AND "Sum of remainingSetCount > 0" AND "(currentDate > + od.expectedDispatchDate"
      1. Show text 'X Delay'; where X = No. of days of Delay i.e (currentTimeStamp - od.expectedDispatchDate). Text Color #FC4B6C
    2. If one or more than one variants of the product has (od.orderStage=1 OR od.orderStage = 2) AND "Sum of remainingSetCount > 0" AND "(currentDate = od.expectedDispatchDate"
      1. Show text 'Dispatch Today'; Text Color: Theme green
    3. If all variants of the product have either ( od.orderStage = 3 OR "Sum of od.remainingSetCount=0") ,
      1. Do not Show any Status. Leave the field blank.
15Order Items > Variant Row

Following are the data points for each variant block:

  1. Variant Color Name (appearing at the top of the variant Image)
  2. Variant image
  3. Act Delay: This Text will remain hidden as long as the od.b2Delay is NULL or Negative. Show this value only when od.b2Delay >0. The value will can be in Minutes, Hours or Days. i.e
    1. Act Delay - X Minutes
    2. Act Delay - X Hours
    3. Act Delay - X Days
  4. Article Code: Show this text only if p.brandCode is NOT NULL. (Character Limit 10)
  5. Variant Status. (Explained Below)
  6. Order Quantity: This is the value of remaining sets of that variant (od.remainingSetCount)
  7. Set Size: This is the Set size ordered by the buyer (od.selSetSize)
  8. Cancellation Status (Explained Below)
  9. This row will become 50% opaque if the "remainingSetCount=0"

20Payment Tab > Master Status

This section will show the summary of order in terms of amounts, pairs and status.

Below is the where conditions for each item:

  • Undelivered: All order items WHERE od.remainingSetCount>0 AND (od.orderStage=1 OR orderStage=2 OR (cm.actDeliveryDate=0 AND cm.rtoDate=0))
  • Cancelled (Buyer): All order items WHERE "cancelledBy = Buyer" in cancellation Log
  • Cancelled (Seller): All order items WHERE "cancelledBy = Seller" in cancellation Log
  • RTO: All order items WHERE cm.rtoDate>0
  • Delivered: All order items WHERE (cm.actDeliveryDate>0 AND cm.rtoDate=0)

The sum of all the above values should match the value of od.netAmount.

How to calculate "Pairs":

  1. for Non - Cancelled items: SUM ( od.setSize * od.setCount )
  2. for Cancelled Items: SUM (od.setSize * cl.setCount)

How to calculate "Value":

  1. For Non - Cancelled Items: SUM ( (od.SKPrice + od.extSKPrice + (o.shipping/o.pairs) + (o.shippingTax/o.pairs) - (o.advDisocunt/o.pairs) - (o.couponDiscount/o.pairs)) * od.setSize * od.setCount )
  2. For Non - Cancelled: SUM ( (od.SKPrice + od.extSKPrice + (o.shipping/o.pairs) + (o.shippingTax/o.pairs) - (o.advDisocunt/o.pairs) - (o.couponDiscount/o.pairs)) * od.setSize * cl..setCount )

Note - All the above values must be inclusive of

  • (add) perPairShipping * no. of pairs (o.shipping/o.pairs)
  • (add) perPairShippingTax * no. of pairs (o.shippingTax/o.pairs)
  • (less) perPairAdvDiscount * no of pairs (o.advDisocunt/o.pairs)
  • (less) perPairCouponDiscount * no. of pairs (o.couponDiscount/o.pairs)
