Documentation : New Orders (old)

Created by Siddharth Rastogi, last modified on Dec 10, 2018

Overview

This screen will be used to receive new orders. From here sellers can move orders to "In Process" Stage.

Data Points and Logic

S.NoData Points / ButtonsLogicStatus
1Summary

Total Orders – Sum of all the orders on this Page.

Total Products – Sum of all the Products in the Orders

Changed
2PaginationNew Design. Has dropdown of no. of pages. On click of any dropdown option, load the particular page.New
3"Move To In Process" Button

This button will remain disabled if no new orders are present.

This Button will be enabled if atleast one new order is Present.

On click, move all the orders present in the "New Orders" Screen to "In Process" Screen and update the "confirmedOn" field withe the Timestamp 

On click, also update the following fields 'in order_details' table. The logic for this can be found in old Admin "Confirm Orders Screen" on click of confirm button. 

  • expectedDispatchDate
  • expectedDeliveryDate. 
Existing
4Order Row

The Orders will appear if the following conditions match:

  • Order Status = 4
  • Atleast one verification should be approved by Admin (either GST or Document)


Each Order Row will contain the following:

  1. Buyer Name (City) [Existing]
  2. Info Icon [New]
  3. SKOrderID [Existing]
  4. Date of placing the order [Existing]
  5. Total Order Value [Existing]
  6. Total Products [Existing]
  7. Total Pairs [Existing]
  8. Cancel Button
Changed
5Info IconThis icon appears next to the Buyer Name (City) on the Order Row. On click, open the Buyer Info Popup. Explained Separately in other page.New
6Cancel Button

On click, Update the Status of the Order [Order Status= 5, Cancelled On, Cancelled By, Cancelled Reason ]

Also, update the Cancellation status at variant level and save the Stage of cancellation. 

Implications:

Send Push and SMS to the Buyer intimating about the cancellation.

____________

"%Buyer’s_company_name%,

The seller %Seller_Company_Name% has cancelled your order ID - %SKOrderID%.

Helpline: %phone%"

New
7Product Row

Each Product Row is collapsible. Each row will contain following:

  1. Brand Name and Counter Code [Existing]
  2. SK Price [Existing]
  3. Total (SK Price (+ ext SK price if any) x no. of pairs)) [Existing]
  4. Order Quantity: Will show Total no. of Lots ordered of that product [New]
  5. No. of Colors: Will show the total no. of Variants of that product ordered [New]
  6. Sell Type: (Carton or Lot) [New]
  7. Dispatch Time – Will show the Dispatch Time in Days units [Changed]
Changed
8Variant Row

This row will expand/collapse on click of the product row. This row contains the following:

  1. Color Name [Existing]
  2. Image Thumbnail [Existing]
  3. Article Code [Existing]
  4. Order Quantity (In Lots) for each variant [Existing]
  5. Size Set [Existing]

Existing

UX Notes - By default, all the orders and their product rows must be visible. User can then toggle  and expand/collapse respective product rows.


Notes For Next Release

Transfer Confirmation responsibility on Seller.

Seller Can cancel orders from this page.

This Page will show all the Orders that have been placed and at least one verification (GST or Document) is complete. Therefore, the orders must appear in the following conditions:
a) GST Value > 0 & GST Verified = 1
& DocumentURL > 0 & DocumentVerified = 1
b) GST Value > 0 & GST Verified = 1
& DocumentURL = 0 & DocumentVerified = 0
c) GST Value = 0 & GST Verified = 0
& DocumentURL > 0 & DocumentVerified = 1

4) “Confirm All Orders” Button– This Button will mark confirm all the Orders in the “New Orders” Tab
5) “Confirm Selected Orders” Button – This button will remain disabled (grayed out) by default. Once any order is checked, this button will enable and start showing the count of selected orders. On click of this button, all the selected orders will be marked confirmed.
