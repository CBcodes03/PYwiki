Documentation : Brand Management System (Fixes)

Created by Siddharth Rastogi, last modified on May 22, 2018

Overview

  • One Brand can have multiple Sellers but only one owner
  • One Seller can sell multiple brands

Table Structure

  • Brand_request - sellerId, brandId, brandLogo, registryDoc, nocDoc, isRegistered, requestDate, requestStatus, remarks
  • New: Brand Logo URL to be saved in 'term_type' table (tt.leadImg)
  • All the Approved Brands gets saved in terms table (termid)
  • All the brand and Seller linkages gets saved in term_type_relationship table

The Seller Board will be responsible to generate brand requests. and the admin will be responsible to Approve or reject the brand request.

Seller Board > My Account > Add Brand Popup

When Seller enters a brand name, check for the following:

  1. If the brand exists. (Check the brand name in brand_request table, WHERE br.brandId IS NOT NULL and br.requestStaus=1)
  2. If the brand logo exists. (Check brand_request table WHERE br.brandId IS NOT NULL and br.requestStatus=1 AND br.brandLogo IS NOT NULL)
  3. If the Brand is Registered/Unregistered (Check isRegistered status for brands WHERE tr.status = 1 AND br.requestStatus=1)
  4. If Brand is unregistered, then do not show any doc uploading fields.
  5. If the brand is Registered, check if Registry Doc is present. 
    1. If present, ask for NOC DOC (mandatory)
    2. If not present, ask for both (one mandatory)


Brand Table Ui 

  • Change the column name from "Status" to "Approval Status" 
  • Show the info icon in Request Status column only when br.remarks is NOT NULL and requestStatus=2 


In Approval Status, show text "Unapproved" instead of Rejected.


If tr.status = 1 then show App Status = ACTIVE

Else, show App Status = INACTIVE


Admin > Brand Management System

Remove the isRegistered Column.


When Admin approves a Brand request.

  1. Remove the brand logo uploading option from brand request tab.
  2. How to check when the brand is unlinked with seller and show the message on Admin and Seller Board?
    1. Linking of sellers from tr
  3. Check the app (web services) implications 


In brand_request table, in "status" column show the following data points:

  1. Listed/New 
  2. If Listed append in brackets "Registered with registryDoc" or "Registered with NOC Doc" or "Unregistered"


In Case a brand request comes with a registry doc for a brand that is already listed with registry doc, then show popup stating "This brand already has an owner. Please contact admin to resolve this issue"

When Admin takes action on this popup, then On click of Yes, 

  • Unlink the existing owners so that app stops showing the brand for that seller
  • Update the brand request of the existing owner. Set resquestStatus=2 and set remarks:
    • "Deactivated due to unavailability of NOC" if new owner has the registryDoc
    • "Deactivated due to NOC/Registry Doc" If the new seller has the NOC Doc


In case when, where the seller is automatically removed because of submission of new genuine docs, then add a remark in the brand request table "Deactivated by Admin" and set requestStatus=2


Insertions:

  1. If Brand logo provided, then save the brandLogoUrl, for all the brand request entries where br.brandLogo IS Null AND brandName = brand name given in Seller Board and br.requestStatus=1.


Updating brand Logo URLs

When Approving Brand request with brand Logo, then update all the approved entries having the same brand name with the brandLogo.

In Approved Tab > When uploading a brandLogo, update that logo url in tr and in brand_request table for all the approved entries having the same brand name. 

________

Change the column label "Status" to "App Status"

In Seller Details Page > Brand Section, when toggling the status button, set the tr.status =0/1. Remove the linkages of App Status and Approval Status.


Additional Requirement

When Approving a Brand Request, run a check on the seller 

If um.isSellerActive = 1, run all the logics to approve the brand

If um.isSellerActive = 0, then do not approve the brand and show error toast "Cannot approve this brand. Please approve the seller first."


In Approved Brands Tab (22-05-2018)

  1. Lock the isRegistered Toggle
  2. In the brand Owner Column, below the owner name, show another very small button "Disable Owner" On click, open popup saying "This will remove the owner of the brand and he/she will not be able to upload any products under this brand. Also, all existing products of the owner for this brand will not be visible on app." On click YES, do the following tasks
    1. set tr.status=0 for the owner and brand
    2. set br.requestStatus=2
    3. update br.reason with text - There are some problems with the provided documents.
