Documentation : In Process Page

Created by Siddharth Rastogi, last modified on Apr 30, 2018

Overview

Here, sellers can browse all the orders and items that are "In Process". Major actionable on this page: 

  1. Move to Packaging
  2. Cancel Lots
  3. Cancel Order
  4. Remove Orders/Products

Screen Preview at the bottom of this page.

Cancel Order

On click, open the popup shown below. Seller will have to select one reason to Cancel the Order. (List of Reasons will be provided soon) 

On Click of "Confirm Cancellation", following Data points should be updated for every lot whose status is "In Process":

  1. Cancelled By - Seller
  2. Cancelled On - TimeStamp
  3. Reason of Cancellation - As given by seller
  4. Cancelled Order Item ID
  5. No. of Lots Cancelled
  6. Status at the time of cancellation

Also, hide that order from this page.

Impact of Cancellation:

a. Send Push Notification and SMS to the buyer about cancellation. Now, check if the no. of lots cancelled is equal to the no. of lots in that order,

  • If yes, then send Push Notification "%seller_name% has cancelled your order - %skorderID%"
  • If No, then send then the push notifications "X lots have been cancelled by the Seller - %seller_name% from your order - %skorderid%     

b. Send SMS to Buyer about cancellation. Use above condition here as well

  • If yes, then send Push Notification "Dear Buyer, Seller - %seller_name% has cancelled your order - %skorderID%. Check 'My Orders' Section in the app for updates."
  • If No, then send then the push notifications "Dear Buyer, %X% lots have been cancelled by the Seller - %seller_name% from your order - %skorderid%. Check 'My Orders' Section in the app for updates."

7B2D Delay Info
  1. If current Date < Dispatch Date, show nothing
  2. If Current Date = Dispatch Date, show text “(Dispatch Today)” Font Color: Green
  3. If Current Date > Dispatch Date, show “(X Days/hours Delay)” Font Color: Red
New
8Move to Packaging Button (Product Row)

On Click, all the lots of the product having status “In Process” will move to “In Packaging”.

-Also check if variants of any other product in that order still has status "In Process"

  • If Not, Hide the whole Order Entry.
  • If Yes, Do nothing
New
9Remove Product Button (Product row)

This button will appear if all the lots having Status “In Process” in that product have been Cancelled. On click, that product entry must hide from that order.

Also,  check if the Order contains any other product whose variant has status "In Process" and not cancelled.

  • If Not, remove the whole order entry as well.
  • If Yes, Remove only the particular product. 
New
10Variant Row

This row will expand/collapse on click of the product row. This row contains the following:

  1. Color Name [Existing]
  2. Image Thumbnail [Existing]
  3. Article Code [Existing]
  4. Order Quantity (In Lots) for each variant excluding cancelled lots [Existing]
  5. Size Set [Existing]
  6. Cancel Option Button. On click, open dropdown with option "Cancel Item". Explained further below. 
  7. Move to packaging button. Explained Below
  8. Cancellation by Buyer/Seller Info. Explained Below

New
11Cancel Item (Variant row)

On click of "Cancel Item", open the popup shown below.

Seller must select the no. of lots to cancel from the dropdown. The Dropdown will contain values in lots of the variant. Minimum Value being '1' & Max. value being the (Lots ordered - Lots Cancelled).

Seller will also select the reason of Cancellation. (List of Reasons will be provided soon) 

On Click of "Confirm Cancellation", following Data points should be updated in cancellation table

  1. Cancelled By - Seller
  2. Cancelled On - TimeStamp
  3. Reason of Cancellation - As given by seller
  4. Cancelled Order Item ID
  5. No. of Lots Cancelled
  6. Status at the time of cancellation

Impact of Cancellation:

  1. Send Push Notification to the buyer about cancellation
  2. Send SMS to Buyer about cancellation
New
12Move to Packaging (Variant row)

On Click, This will move all the Lots of this variant (excluding cancelled) to “In Packaging”. 

On click of this button, also check for statuses of other lots in that order. If no other lots have status “In Process”, then automatically remove the Order entry 

New
13Cancellation by Buyer/Seller info

There can be two Cancelled statuses for each variant (one for buyer and one for seller), therefore cgroup the cancellations by Buyer and Seller Separately.

On hovering on this status, it will the show the reason of cancellation in a tooltip. As we will on club and show the cancellations by Buyers and Sellers there could be more than one reason for cancellation for each. Hence, in this case show multiple reasons on hovering.

If all the Lots of any variant gets cancelled, then the whole variant block will become translucent to 50% and “options” button and “move to packaging: button will be disabled. (Although, the cancellation tooltip should still work)

New

Other Notes* - Add "ProductId" field in the Order Details Table.

Checks for Cancellation

To determine whether we can mark order status as cancelled or not

Validation for Order Cancel and Hiding Order Entry, On Cancellation at Lot Level,

Check whether all the other variants in that order are marked Cancelled or not,(i.e. all the remaining qty of all variants in that Order = 0)

  • If Yes, then update cancellation mark at the Order Level as well [Status = 5, Cancelled By, Cancelled On] and remove the order row.  
  • If No, then check if all the other variants in that order that have status "In Process" are marked Cancelled or Not.
    • If Yes, then hide that order entry from In process Page
    • If No, then check if all the other Variants in that product having status "In Process" been cancelled or not:
      • If Yes, then hide the product entry
      • If No, then just cancel the given variant and make the variant block translucent to 50% where all the options should get locked except the cancellation tooltip should remain working)
