Documentation : Module Level App Update

Created by Siddharth Rastogi, last modified on Jul 18, 2018

Objective

The app should have some special modules where it can prompt the user to update the App in order to use the particular module.

The prompt shall be a popup which will display a message for updating the app and will have two Buttons:

  1. Close - This shall close the popup and take user back to previous page
  2. Update App - This shall open the App link to the Playstore 

Implementation

Module level update will be set in following modules of the App:

  1. My Truck
  2. Add to Truck
  3. Return Refunds
  4. My Orders
  5. Track Consignments 

Each module will have special API parameters that shall run this feature:

  • status
  • dynamic message
  • Build Version

For Force Update, status = 99

For Moderate Update, status = 101


Global Level Updates

  • For keeping App in Maintenance mode, GetStaticData API (runs on Splash Screen) would return status = 100 with dynamic message. In this case, app would show the message with one button. This button would close the app. 
  • Similary, if Api returns status = 99 in GetStaticData API, App will show a Force Update Popup having one button "Update App'
  •  API could also return status = 101 with dynamic for moderate update. In this case, app would show message with 2 buttons. One to skip the dialogue and continue with app, second to go to PlayStore to update the app
