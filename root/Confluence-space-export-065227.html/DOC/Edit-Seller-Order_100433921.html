Documentation : Edit Seller Order

Created by Siddharth Rastogi, last modified on May 07, 2018

Overview

This page will open on click of "Edit Order" button from My Truck Page. This page will basically help a buyer to alter product items in case buyer wants to make changes or in case of some product moq or seller moq violations. 
This page will show following details of a single order. 

  1. Seller Details
  2. Product Details

The page title will be "Edit Seller Order"

Data Points and Logics

#Data Points / ButtonsLogic 
1Seller Name Company Name of the Seller
2Seller MOQ

Show the Seller MOQ Value below the Seller Name. 

In case the value = 0, show text "No Seller MOQ"

3Add Products Button This button appears to the right of the Seller Name. On click, open the seller shop page of that seller.
4Order Value and Pairs

This will show the sum value of all products in that order [Sum of (SK Price x Pairs)]. This will be appended by the count of total pairs ordered. The color of the text will change according to following condition:

  • If Pairs Ordered < Seller MOQ, set the color as Red #FF6565
  • If the Pairs Ordered ≥ Seller MOQ, set the color as Green #00C29C
5Info text Box

This is a red info box that appears above the Order Value text. This box will display texts to the user in case there is a Seller MOQ or Product MOQ violations. This box will appear only in two cases:

Case 1: If the no. of pairs of any product in an order becomes less than the Product MOQ. This could be because any product became inactive (isAvailable = 0) or is StockLot of any variant become 0 or brand is Deactivated. etc..

  • Show text: "EDIT ORDER: X Product's MOQ does not match the order qty" ; where X = Count pf products which have this violation.

Case 2: If the no. of pairs of any order is less than the sellerMOQ. 

  • Show text: "Add X more pairs to complete Seller MOQ" ; where X is the difference of pairs between pairs ordered and sellerMOQ

Case 3: In all other cases do not show this box. 

6Product ListShow list of all the products (group variants by product id) in that order.
7Product Name Show the Brand Name & Brand Counter. 
8Product value and pairs

Show this data point on the top right corner of each card. Show amount of Sum value of all variants of a product [Sum of (SK Price x Pairs)]. Appended by Sum of pairs of that product.

9Expected Delivery Days

Show the text:

"Expected Delivery in X Days" ; where X = Delivery Time of that product in Days 

10Product Violation info text

Show the text below the expected Delivery text. Show this text only when, the valid pairs ordered of a product is less than the product MOQ.

Text: "Add X Pairs to complete the Product MOQ of Y Pairs"

where; X = the value of difference between the product MOQ and the Sum of pairs ordered of that product.

and Y = Value of Product MOQ

10Variant Thumb Image

Show the list of all variant thumb images of that product. 

In case the product is no more available. apply a black overlay (50%) and show text "OUT OF STOCK"

The above case can come in following conditions.

  1. v.isAvaiable=0
  2. Brand for that seller is Deactivated
  3. Stock Lot of variant has become 0

Below each image show the no of pairs of each variant.

11Edit Product Button 

This button appears at the bottom of the card. On click, open the Add to Truck Page in EDIT Mode.

12Remove Product Button

On click, open popup:

Main Text: "Are you sure you want to remove this product ?"

Options: No, Yes

On click Yes, delete the order details entry of all the variants of that product so that it stops showing in the list.

On Cancelling, check if there are no order items left in that order. In that case, delete the Order Entry as well.
