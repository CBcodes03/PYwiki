Documentation : Buyer Verification (Removed)

Created by Siddharth Rastogi, last modified on May 15, 2018

Overview

This screen will be used to verify the buyers who have placed an order but their documents are not verified. Since a buyer can place multiple orders at once, this page will show a combined order entry on the basis of the time of order placed. For example: if a buyer did a multiple checkout on 3 sellers, a combined entry will be shown on this page.

Overall, admin can do 3 actions here:

  • Cancel the Order
  • Verify the Buyer
  • Change the Address

Logic to show Orders

Only those orders will be visible here, whose buyer has his documents pending for verification and order is not confirmed (o.status =4 AND o.confirmedOn=0).

For every order check, 'tinNo' field in user_master table.

  • If Empty, check 'orderVerifyDone'
    • If orderVerifyDone = 1, then do not show the order entry
    • If orderVeridyDone = 0, then show the order entry
  • If Not Empty, check 'isGSTVerified'
    • if isGSTVerified = 1, then do not show the order entry
    • if isGSTVerified = 0, then show the order entry

This page will have the standard sort, filter and Search features. (Do not show the Download CSV button)

Data Points and Logic

#Data Points / ButtonsLogic
1Total Buyers

This appears below the Search, Filter section. This will show the total count of orders that appear on this page. (logic is same as explained above).

On applying Search and filters, this data should change accordingly.

2PaginationThis is the standard pagination used for all other pages.
3R / N icons 

These icons appear before the Buyers name in Order Card. R stands for Repeat Buyer and N Stands for New Buyer.

For any order only one of these icons will be present at a given time. 

Logic to show Repeat/New icons

Show R Icon: if the buyer has placed 2 or more orders and atleast one order has o.status=4 AND o.confirmedOn>0

Show N icon: in else condition of the above logic.

Keep orange color for R and Cyan color for N.

4Buyer Name (City)

The 'companyName' of the buyer who has placed the order. Appended by the city to which he belongs. Fetch the city from the order table "delCity". 

Maximum character limit of 45. If exceeds show ....

5Buyer Phone No.This appears just below the Buyer Name. 'companyPhone'
6Verification info 

This section appears to the right of the buyer name. There will be two labels:

  1. Document Verification
  2. GST Verification

Each label will be appended by either Red exclamation icon, Green Tick icon or 'Not Provided' text on the basis of the following condition.

Logic for verification icon

For Document Verification label,

  • Show Red Exclamation icon if, orderVerifyDoc is not NULL AND orderVerifyDone = 0
  • Show Green Tick icon if, orderVerifyDoc is not NULL AND orderVerifyDone = 1
  • Show 'Not Provided' text if, if, orderVerifyDoc is NULL 

For GST Verification Label, 

  • Show Red Exclamation icon if, tinNo is not NULL AND isGSTVerified=0
  • Show Green Tick icon if, tinNo is not NULL AND isGSTVerified=1
  • Show 'Not Provided' text if, if, isGSTVerified is NULL 
7Order Placed DateShow the date of placing the order. Fetch this from ''updatedOn"  field in Orders table.
8Order Total Amounts and Pairs

Show the Sum of  Net Value of the Orders placed at same placed Date (Sum of "netAmount" from Orders Table).

Append this with total SKUs of all orders placed on same placed date (Sum of "items" from Orders Table)

Append this with total pairs of all orders placed on same placed date (Sum of "pairs" from Orders Table)

Each Order Card will have three tab menus. 

  • Verification and Payment

  • Comment Log (x) - Not there in Design

On clicking any menu, the card will expand to their respective Ui's. On clicking again on an already opened menu tab, will collapse the card to its default Ui. 

10Verification and Payment Menu Tab

This Tab contains three sections:

  • Document Verification
  • GST Verification
  • Payment Details

(Each Explained Below)

11Document Verification

This section has following items:

  1. Verification Type : This will show a dropdown showing the values selected by the buyer from the app. ("orderVerifyType" from orders table). On clicking the dropdown a list of options will appear: Bill Book, Visiting Card and Shop Photo. On selection of any option it will set the value in the dropdown. (do not update in database at this stage). if the field is empty, show empty dropdown.
  2. Verification info: If the 'orderVerifyDoc' field is not NULL, show a hyperlinked text "View Image" on click of which open the url file in a new tab. In case the 'orderVerifyDoc' field is NULL, show a text "No Image"
  3. Checkbox - This check box will appear checked if "orderVerifyDone = 1" else unchecked. Admin will click on this checkbox and click on "Mark Verified" Button to update the verification status of the document.
12GST Verification

This section has following items:

  1. GST Number: this will show an edit box which will fetch values from 'tinNo' from user_master table.
  2. Checkbox - This check box will appear checked if "isGSTVerified= 1" else unchecked. Admin will click on this checkbox and click on "Mark Verified" Button to update the verification status of the GST.
13Payment Details 

This will show the payment details as a sum value of all the orders placed on the same date and time.

  • Total Pairs (Sum of 'pairs' orders table)
  • Subtotals (Sum of od.setsize*od.setcount*(od.Skprice+od.extSKPrice) of each order item from order_details table)
  • Shipping Charge (Sum of shipping+shippingTaxAmt)
  • Coupon Discount (Sum of couponDiscount)
  • Advance Discount (Sum of advDiscount)
  • Net Amount (Sum of netAmount)
  • Payment Option (paymentOption)
  • Payment Mode (paymentMode)
  • Advance (Sum of advnace)
  • Balance (Sum of balance)

15Shipping Address Popup

This popup will show the pre-filled address information of the delivery address given by buyer. This popup will be used to edit the address. On Updating the address all the orders placed on same placed date must get updated.

  • Street Address (o.delDtreetAddress) - new 
  • Pincode (o.delPincode)
  • Check Availability Button - On click, run the pincode servicibility api used in the app and find if the pincode is available or not. If available, show green tick, if not available show red exclamation mark.
  • City and State fields will be locked edit boxed. By default they will be prefilled and fetched from the o.delcity and o.delstate from orders table. They will be updated by the servicibility api response as well.
  • Update Button: on click, Check the Servicibility api for pincode abailability. If yes, update the pincode, city, state and street address in orders table. If Not, show exclamation mark next to pincode edit box. 
15Sellers menu tab

This will expand and show the list of sellers on whom the order was placed.

  • Name of Seller 
  • Seller's city
  • OrderID
  • Net Amount | Total SKU | Total pairs - for individual sellers' orders
16Comment log

The Comment Log text will be appended by "(X)" ; where X = no. of comments on that user. 

On click, open the comment log popup (as designed by Pratyush). This will fetch and update data from the user_master table.

16Cancel Order

On click, update the status of all the orders having same placed date as o.status = 5 AND update o.cancelledOn with the timestamp AND isRemoved=1

Also, update the following information at the Order Details level and update the entries in CancellationLog Table.

remainingSetCount=0, isRemoved=1.

Do no show this button is (isGSTVerified=1 OR orderVerifyDone =1)

17Mark Verified

On Click, update the information provided in the Documents Verification and GST Verification Section.

After updating,

  • if still any verification is pending, then show toast message "Unable to verify as Buyer Verification is incomplete."
  • If no verification is pending, then show toast message "Buyer is successfully verified "
  • If buyers' tinNo > 0 and isGSTVerified =1, then update the tinNo. in Orders Table as well.
